Run it on your own machine

≈ 15 min
Install Docker Desktop and Git
Docker Desktop (Windows/Mac) or Docker Engine (Linux), plus Git. That's all you need — Node, PostgreSQL and MongoDB all run inside containers.
Clone the repository and switch to the project branch
git clone https://github.com/rchezhian81-prog/SREEDO.git
cd SREEDO
git checkout claude/tender-einstein-fdks9c
Configure environment and start everything
cp .env.example .env   # edit: set POSTGRES_PASSWORD + both JWT secrets
docker compose up --build
First build takes a few minutes. Postgres, MongoDB, the API, the web app and nginx all start together.
Log in and click through every page
Open http://localhost and sign in with admin@sreedo.edu / Admin@12345. Try students, attendance marking, fee invoicing, announcements. API docs live at http://localhost/api/docs.
💡 This step matters most — anything you want changed is far cheaper to fix now than after deployment.
2

Merge to main so CI protects you

≈ 10 min
Open a pull request and merge the branch into main
On GitHub: Compare & pull request from claude/tender-einstein-fdks9c → main, then merge. (Or ask Claude to open the PR for you.)
Watch the first GitHub Actions run go green
CI runs on every push: backend typecheck + tests + build, frontend build, flutter analyze, Docker image builds. The Flutter job is the one piece not yet verified locally — if it flags anything, it's a quick fix.
3

Enable the optional integrations

≈ 20 min
AI assistant — set OPENAI_API_KEY
Get a key at platform.openai.com, add OPENAI_API_KEY=sk-… to .env, restart. The Assistant page starts answering with live school data instantly — no code changes.
Email receipts — set SMTP credentials
Hostinger email hosting gives you SMTP. Set SMTP_HOST / SMTP_PORT / SMTP_USER / SMTP_PASS in .env; payment receipts to guardians then send automatically.
4

Mobile app

≈ 1–2 hrs
Install Flutter and generate platform folders
cd mobile
flutter create . --platforms android,ios --project-name sreedo_mobile
flutter pub get
flutter run --dart-define=API_URL=http://10.0.2.2:4000/api/v1
10.0.2.2 is how the Android emulator reaches your computer's localhost.
Optional: enable push notifications
Create a Firebase project, then run flutterfire configure. The app already handles both configured and unconfigured states.
5

Deploy to your Hostinger VPS

≈ 1–2 hrs
Point a domain at the VPS
Add an A record for your domain → the VPS IP address.
SSH in, install Docker, clone the repo, create the production .env
openssl rand -hex 64   # run twice — one for each JWT secret
Also set a strong POSTGRES_PASSWORD, NEXT_PUBLIC_API_URL=https://yourdomain.com/api/v1 and CORS_ORIGIN=https://yourdomain.com.
Launch and add TLS
docker compose up -d --build
Confirm the site loads over HTTP, then issue certificates with certbot (certbot --nginx) — the TLS path is documented in infra/nginx/default.conf.
Lock down the seeded admin account
Immediately after the first boot: log in, change the admin password, set SEED_ON_START=false in .env, restart.
⚠️ Never leave Admin@12345 live on a public server — change it within minutes of deploying.
6

Go-live checklist — don't skip

≈ half a day
Set up nightly database backups
# example cron entry (2:00 AM daily, keep 14 days)
0 2 * * * docker exec sreedo-postgres-1 pg_dump -U sreedo sreedo | gzip > /backups/sreedo-$(date +\%F).sql.gz
Ship copies off the VPS (e.g. Hostinger object storage). This is the single most important operational task — fee records are money.
Load real school data
Create your academic year, classes & sections, subjects, teachers, then students — the web UI covers all of it.
Create real staff accounts with proper roles
Use the Users page to create teacher and accountant accounts instead of sharing the admin login.
